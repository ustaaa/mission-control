## ðŸ”® Future: Native App Wrapping

For app store distribution, consider:

### Android - TWA (Trusted Web Activity)
```bash
# Using Bubblewrap CLI
npx @anthropic-ai/anthropic-sdk -g @nicknisi/bubble-wrap
bubblewrap init --manifest https://your-domain.com/manifest.json
bubblewrap build
# Outputs APK for Google Play
```

### iOS/Android - Capacitor
```bash
npm install @capacitor/core @capacitor/cli
npx cap init "Mission Control" com.moltbot.missioncontrol
npx cap add ios
npx cap add android
npx cap sync
# Opens in Xcode/Android Studio
```

### Considerations
- Native wrappers add ~2-5MB overhead
- App store review process required
- Updates require new builds (not instant like PWA)
- PWA is sufficient for most use cases

### "Add to Home Screen" not appearing?
- Must be served over HTTPS (or localhost)
- Manifest must be valid JSON
- Service worker must be registered

### Offline not working?

- Clear cache and reload: `navigator.serviceWorker.getRegistrations().then(r => r.forEach(sw => sw.unregister()))`
